# Vimeo

Vimeo Webstudio component.

## Installation

Use the Webstudio component registry item `vimeo`.

In Builder automation, resolve the item from the generated shadcn-compatible registry and use the Webstudio metadata in `meta` for insertion.

Prefer the `Vimeo` template when creating a new instance from scratch.

## Usage

Use this component through Webstudio's instance, props, styles, resources, and data-source model.

Inspect the registry metadata before creating or editing instances so content model, states, and composition stay valid.

## Composition

Use the following composition to build `Vimeo`:

```text
Vimeo
|-- VimeoSpinner
|-- VimeoPlayButton
`-- VimeoPreviewImage
```

## Examples

Use the `Vimeo` template as the canonical starter. Add a video to your page that is hosted on Vimeo. Paste a Vimeo URL and configure the video in the Settings tab.

## API Reference

### Props

- `autopause`: type: `boolean`, control: `boolean`, default: `true`. Whether to pause the current video when another Vimeo video on the same page starts to play. Set this value to false to permit simultaneous playback of all the videos on the page. This option has no effect if you've disabled cookies in your browser, either through browser settings or with an extension or plugin.
- `autopip`: type: `boolean`, control: `boolean`. Whether to enable the browser to enter picture-in-picture mode automatically when switching tabs or windows, where supported.
- `autoplay`: type: `boolean`, control: `boolean`, default: `false`. Whether to start playback of the video automatically. This feature might not work on all devices.
  Some browsers require the `muted` parameter to be set to `true` for autoplay to work.
- `backgroundMode`: type: `boolean`, control: `boolean`. Whether the player is in background mode, which hides the playback controls, enables autoplay, and loops the video.
- `controlsColor`: type: `string`, control: `color`. A color value of the playback controls, which is normally #00ADEF. The embed settings of the video might override this value.
- `doNotTrack`: type: `boolean`, control: `boolean`, default: `false`. Whether to prevent the player from tracking session data, including cookies. Keep in mind that setting this argument to true also blocks video stats.
- `interactiveParams`: type: `string`, control: `text`. Key-value pairs representing dynamic parameters that are utilized on interactive videos with live elements, such as title=my-video,subtitle=interactive.
- `keyboard`: type: `boolean`, control: `boolean`, default: `true`. Whether to enable keyboard input to trigger player events. This setting doesn't affect tab control.
- `loading`: type: `string`, control: `radio`, default: `"lazy"`. Not a Vimeo attribute: Loading attribute for the iframe allows to eager or lazy load the source
- `loop`: type: `boolean`, control: `boolean`, default: `false`. Whether to restart the video automatically after reaching the end.
- `muted`: type: `boolean`, control: `boolean`, default: `false`. Whether the video is muted upon loading. The true value is required for the autoplay behavior in some browsers.
- `pip`: type: `boolean`, control: `boolean`, default: `false`. Whether to include the picture-in-picture button among the player controls and enable the picture-in-picture API.
- `playsinline`: type: `boolean`, control: `boolean`, default: `false`. Whether the video plays inline on supported mobile devices. To force the device to play the video in fullscreen mode instead, set this value to false.
- `quality`: type: `string`, control: `select`, default: `"auto"`. For videos on a Vimeo Plus account or higher: the playback quality of the video. Use auto for the best possible quality given available bandwidth and other factors. You can also specify 360p, 540p, 720p, 1080p, 2k, and 4k.
- `responsive`: type: `boolean`, control: `boolean`, default: `true`. Whether to return a responsive embed code, or one that provides intelligent adjustments based on viewing conditions. We recommend this option for mobile-optimized sites.
- `showByline`: type: `boolean`, control: `boolean`, default: `false`. Whether to display the video owner's name.
- `showControls`: type: `boolean`, control: `boolean`, default: `true`. Whether to display the player's interactive elements, including the play bar and sharing buttons. Set this option to false for a chromeless experience. To control playback when the play/pause button is hidden, set autoplay to true, use keyboard controls (which remain active), or implement our player SDK.
- `showPortrait`: type: `boolean`, control: `boolean`, default: `true`. Whether to display the video owner's portrait. Only works if either title or byline are also enabled
- `showPreview`: type: `boolean`, control: `boolean`, default: `false`. Not a Vimeo attribute: Whether the preview image should be loaded from Vimeo API. Ideally don't use it, because it will show up with some delay and will make your project feel slower.
- `showTitle`: type: `boolean`, control: `boolean`, default: `false`. Whether the player displays the title overlay.
- `speed`: type: `boolean`, control: `boolean`, default: `false`. Whether the player displays speed controls in the preferences menu and enables the playback rate API.
- `texttrack`: type: `string`, control: `text`. The text track to display with the video. Specify the text track by its language code (en), the language code and locale (en-US), or the language code and kind (en.captions). For this argument to work, the video must already have a text track of the given type; see our Help Center or Working with Text Track Uploads for more information.
  To enable automatically generated closed captions instead, provide the value en-x-autogen. Please note that, at the present time, automatic captions are always in English.
- `transparent`: type: `boolean`, control: `boolean`, default: `true`. Whether the responsive player and transparent background are enabled.
- `url`: type: `string`, control: `text`. The ID or the URL of the video on Vimeo. You must supply one of these values to identify the video. When the video's privacy setting is Private, you must use the URL, and the URL must include the h parameter. For more information, see Vimeo’s introductory guide.
