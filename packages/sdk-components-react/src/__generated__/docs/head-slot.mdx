# Head slot

Inserts children into the head of the document

## Installation

Use the Webstudio component registry item `head-slot`.

In Builder automation, resolve the item from the generated shadcn-compatible registry and use the Webstudio metadata in `meta` for insertion.

Prefer the `HeadSlot` template when creating a new instance from scratch.

## Usage

Use this component through Webstudio's instance, props, styles, resources, and data-source model.

Inspect the registry metadata before creating or editing instances so content model, states, and composition stay valid.

## Composition

Use the following composition to build `HeadSlot`:

```text
HeadSlot
|-- HeadLink
|-- HeadMeta
`-- HeadTitle
```

## Examples

Use the `HeadSlot` template as the canonical starter. The Head Slot component lets you customize page-specific head elements (like canonical URLs), which merge with your site's global head settings, with Head Slot definitions taking priority over Page Settings. For site-wide head changes, use project settings instead.
